Research & Frameworks

Oleh Hendri Rahmat Hendrianto • Diperbarui 29 Jun 2026

Halaman ini mendokumentasikan berbagai penelitian dan framework yang saya kembangkan dalam perjalanan blockchain dan AI engineering. Setiap entri mencakup konsep dasar, motivasi, dan status pengembangan terkini.

R-think

R-think adalah framework reframing berpikir yang saya gunakan dalam pemecahan masalah engineering — khususnya saat menghadapi sistem kompleks yang tidak bisa diselesaikan dengan pendekatan linear. Framework ini lahir dari observasi bahwa banyak kegagalan teknis berasal dari frame problem: kita mencoba memecahkan masalah dalam batasan asumsi yang terlalu sempit.

Konsep inti: R-think mendorong recursive reframing — setiap layer solusi harus diuji terhadap asumsi di layer sebelumnya. Jika terjadi deadlock di layer implementasi, jangan perbaiki kode, tapi perbaiki model mental yang melandasinya.

Status: Aktif digunakan dalam pengembangan KVP Protocol Phase 2 dan arsitektur Pevi AI Assistant. Dokumentasi formal sedang dalam penyusunan.

POTT (Proof of Thought and Trust)

POTT adalah framework konsensus hybrid yang menggabungkan verifikasi komputasional (Proof of Thought) dengan reputasi berbasis kepercayaan (Trust). Tidak seperti PoW yang boros energi atau PoS yang bias pada akumulasi modal, POTT mencoba menyeimbangkan partisipasi intelektual dengan akuntabilitas sosial.

Bagaimana POTT bekerja: Node mengajukan thought proof — bukti bahwa mereka telah melakukan komputasi atau analisis bernilai untuk jaringan. Bobot suara node ditentukan oleh kombinasi kualitas thought proof yang pernah diajukan dan skor trust dari peer. Tidak ada stake token — stake-nya adalah reputasi dan kontribusi intelektual.

Status: Tahap konseptual. Masih dalam pengembangan model formal dan simulasi awal.

POD (Proof of Development)

POD adalah mekanisme yang memungkinkan developer membuktikan kontribusi kode dan arsitektur mereka secara on-chain tanpa harus mengekspos IP atau kode sumber. POD memanfaatkan zero-knowledge attestation untuk memvalidasi bahwa kontribusi tertentu telah diberikan — tanpa mengungkapkan detail implementasi.

Use case primer: POD ditujukan untuk DAO development dan open-source funding yang terverifikasi. Alih-alih mempercayai klaim kontribusi secara verbal, DAO dapat memverifikasi bukti kriptografis bahwa work telah dilakukan, tanpa harus membuka repositori privat.

Status: Penelitian aktif. Prototipe attestation layer sedang dikerjakan di atas KVP Protocol infrastructure.

KrakenWorld Experiment

KrakenWorld Experiment adalah proof-of-concept untuk fully on-chain 3D MMORPG dengan governance dan ekonomi yang berjalan sepenuhnya di blockchain. Konsep ini menguji batasan on-chain state management untuk aplikasi real-time dan interaktif.

  • 3 civilization tokens: VEL, ELX, KRT — masing-masing mewakili faksi ekonomi dalam game
  • On-chain DEX: Pemain dapat melakukan swap antar token faksi dan KVC di dalam game
  • Governance: Keputusan game (balance patch, event, reward distribution) diputuskan melalui on-chain voting
  • Settlement bridge: Transaksi in-game diselesaikan melalui KVP Protocol untuk finalitas ekonomi

Pertanyaan riset: Bisakah ledger blockchain menangani state yang berubah setiap detik tanpa mengorbankan finalitas atau biaya transaksi yang wajar? KrakenWorld Experiment adalah laboratorium untuk menjawab pertanyaan ini.

Status: Blueprint stage. Ekonomi game dan tokenomics telah dirancang; implementasi on-chain menunggu KVP Protocol Phase 2.

ACC Discovery Experiment

ACC Discovery Experiment adalah eksperimen untuk mengeksplorasi autonomous code discovery — kemampuan AI agent untuk secara mandiri menemukan, memahami, dan mendokumentasikan arsitektur kode dalam repositori besar tanpa supervisi manusia. Eksperimen ini dijalankan menggunakan Pevi AI Assistant sebagai engine.

Metodologi: ACC Discovery memanfaatkan RAG (Retrieval-Augmented Generation) dengan vector index dari seluruh codebase KVP Protocol, kemudian menjalankan agent loop yang secara otonom: (1) memindai struktur direktori, (2) mengidentifikasi pola arsitektur, (3) menghasilkan dokumentasi teknis, (4) memvalidasi temuannya terhadap eksekusi aktual.

Temuan awal: Agent mampu mengidentifikasi 85% pola arsitektur dan ketergantungan antar modul tanpa panduan manusia. Dokumentasi yang dihasilkan telah digunakan sebagai basis untuk migrating beberapa modul KVP dari Go ke Rust.

Status: Eksperimen berjalan; hasil dipublikasikan secara terbatas. Berencana merilis whitepaper metodologi pada Q3 2026.

Research & Frameworks

By Hendri Rahmat Hendrianto • Updated Jun 29, 2026

This page documents various research projects and frameworks I have developed throughout my blockchain and AI engineering journey. Each entry covers the core concept, motivation, and current development status.

R-think

R-think is a reframing framework I use for engineering problem-solving — especially when facing complex systems that cannot be solved with linear approaches. This framework was born from the observation that many technical failures stem from the frame problem: we try to solve problems within overly narrow assumptions.

Core concept: R-think encourages recursive reframing — each solution layer must be tested against the assumptions of the previous layer. If a deadlock occurs at the implementation layer, don't fix the code — fix the mental model underlying it.

Status: Actively used in KVP Protocol Phase 2 development and Pevi AI Assistant architecture. Formal documentation is being prepared.

POTT (Proof of Thought and Trust)

POTT is a hybrid consensus framework combining computational verification (Proof of Thought) with trust-based reputation (Trust). Unlike energy-wasteful PoW or capital-biased PoS, POTT attempts to balance intellectual participation with social accountability.

How POTT works: Nodes submit a thought proof — evidence that they have performed computation or analysis of value to the network. A node's voting weight is determined by a combination of the quality of thought proofs they have submitted and their trust score from peers. No token staking — the stake is reputation and intellectual contribution.

Status: Conceptual stage. Formal model and initial simulations are in development.

POD (Proof of Development)

POD is a mechanism that allows developers to cryptographically prove code and architecture contributions on-chain without exposing IP or source code. POD uses zero-knowledge attestation to validate that specific contributions were made — without revealing implementation details.

Primary use case: POD is designed for DAO development and verifiable open-source funding. Instead of trusting contribution claims verbally, DAOs can verify cryptographic proof that work was done, without opening private repositories.

Status: Active research. Prototype attestation layer is being built on top of KVP Protocol infrastructure.

KrakenWorld Experiment

KrakenWorld Experiment is a proof-of-concept for a fully on-chain 3D MMORPG with governance and economy running entirely on the blockchain. This concept tests the limits of on-chain state management for real-time interactive applications.

  • 3 civilization tokens: VEL, ELX, KRT — each representing an in-game economic faction
  • On-chain DEX: Players can swap faction tokens and KVC within the game
  • Governance: Game decisions (balance patches, events, reward distribution) decided through on-chain voting
  • Settlement bridge: In-game transactions settled via KVP Protocol for economic finality

Research question: Can a blockchain ledger handle state that changes every second without sacrificing finality or reasonable transaction costs? The KrakenWorld Experiment is the laboratory to answer this question.

Status: Blueprint stage. Game economy and tokenomics have been designed; on-chain implementation awaits KVP Protocol Phase 2.

ACC Discovery Experiment

ACC Discovery Experiment is an exploratory initiative investigating autonomous code discovery — the ability of an AI agent to independently discover, understand, and document code architecture in large repositories without human supervision. This experiment uses Pevi AI Assistant as the engine.

Methodology: ACC Discovery leverages RAG (Retrieval-Augmented Generation) with a vector index of the entire KVP Protocol codebase, then runs an agent loop that autonomously: (1) scans the directory structure, (2) identifies architectural patterns, (3) generates technical documentation, (4) validates its findings against actual execution.

Initial findings: The agent was able to identify 85% of architectural patterns and module dependencies without human guidance. The generated documentation has been used as a basis for migrating several KVP modules from Go to Rust.

Status: Experiment ongoing; results published on a limited basis. Planning to release a methodology whitepaper in Q3 2026.
