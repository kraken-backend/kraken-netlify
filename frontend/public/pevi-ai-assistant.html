Di tengah banyaknya platform AI yang tersedia — dari ChatGPT hingga Claude hingga Gemini — saya memutuskan untuk membangun AI assistant saya sendiri. Bukan karena yang lain tidak bagus, tapi karena saya ingin sepenuhnya independen dan tidak tergantung pada platform manapun untuk mendukung pekerjaan engineering sehari-hari saya.

Filosofi di balik Pevi: Kalau saya bisa membangun Layer-1 blockchain sendiri, saya juga bisa membangun AI assistant sendiri. Pevi bukan hanya tools — ini adalah bukti bahwa setiap sistem yang saya butuhkan, saya bisa bangun sendiri.

Apa itu Pevi?

Pevi adalah personal AI assistant berbasis web yang saya bangun dengan Next.js (App Router), Tailwind CSS, dan SQLite via Prisma ORM. Pevi menggunakan OpenRouter API untuk mengakses 370+ model AI — semuanya gratis, tanpa biaya tambahan.

Fitur Utama

🤖

Multi-Model AI Chat

370+ model via OpenRouter. Auto model selection — Pevi pilih model terbaik berdasarkan konteks (vision, text, kompleksitas).

📄

RAG Document Intelligence

Upload PDF, DOCX, XLSX, PPTX, CSV, dan lainnya. Pevi ekstrak teks, buat vector embedding, dan inject ke context otomatis.

🖼️

Vision Support

Upload gambar atau paste dari clipboard. Auto-deteksi vision model. Base64 di-strip dari history agar context tidak overflow.

💻

Artifact Panel

Setiap kode yang di-generate AI otomatis tampil di sidebar kanan. Code view + live HTML preview. Multi-artifact dengan tab navigator.

📁

File Generation

AI bisa generate DOCX, XLSX, PPTX yang langsung bisa didownload. Output lewat format JSON yang terstruktur.

🔄

Multi-Batch Fallback

Kalau model pertama gagal, Pevi otomatis coba batch berikutnya. Fallback ke OpenCode free models kalau semua OpenRouter gagal.

Tech Stack

Pevi: The Private AI Assistant I Built for Myself

By Hendri Rahmat Hendrianto • 2026

▶ Watch Preview on YouTube

With so many AI platforms available — from ChatGPT to Claude to Gemini — I decided to build my own AI assistant. Not because the others aren't good, but because I wanted to be completely independent and not reliant on any platform to support my day-to-day engineering work.

The philosophy behind Pevi: If I can build a Layer-1 blockchain from scratch, I can build my own AI assistant too. Pevi isn't just a tool — it's proof that any system I need, I can build myself.

What Is Pevi?

Pevi is a web-based personal AI assistant I built with Next.js (App Router), Tailwind CSS, and SQLite via Prisma ORM. Pevi uses the OpenRouter API to access 370+ AI models — all free, at no additional cost.

Key Features

🤖

Multi-Model AI Chat

370+ models via OpenRouter. Auto model selection — Pevi picks the best model based on context (vision, text, complexity).

📄

RAG Document Intelligence

Upload PDF, DOCX, XLSX, PPTX, CSV, and more. Pevi auto-extracts text, creates vector embeddings, and injects context automatically.

🖼️

Vision Support

Upload images or paste from clipboard. Auto-detects vision models. Base64 is stripped from history to prevent context overflow.

💻

Artifact Panel

Any code generated by AI automatically appears in a right sidebar. Code view + live HTML preview. Multi-artifact with tab navigator.

📁

File Generation

AI can generate DOCX, XLSX, PPTX files ready for immediate download, output via structured JSON format.

🔄

Multi-Batch Fallback

If the first model fails, Pevi automatically tries the next batch. Falls back to OpenCode free models if all OpenRouter attempts fail.

Tech Stack

Next.js 16 (App Router) TypeScript Tailwind CSS SQLite + Prisma ORM OpenRouter API Zustand shadcn/ui pdf2json mammoth pptxgenjs react-markdown

Why Did I Build It?

  • Full independence — not tied to any platform's subscription or rate limits
  • Privacy — my engineering data and documents don't leave to third-party servers
  • Customization — I can add features exactly as my own workflow demands
  • KVP integration — future plans include direct integration with KVP Protocol for AI-driven chain operations
Status: Pevi currently runs locally as my personal tool. There are no plans for a public release in the near future, unless an investor or partner is interested in developing it further. Contact me if interested.
